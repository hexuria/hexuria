# Binary Stack

Binary is a built-in pay plan family in the platform.

It is not treated as a later external plugin. It is part of the app architecture and can be used by companies/packages that choose a Binary stack.

## Modules

1. Binary Tree
2. Binary Volume
3. Binary Pairing Bonus
4. Binary Carryover
5. Binary Caps and Rank Rules

## Binary Tree

Places enrollments into left and right legs.

Placement strategies:

- manual left/right
- sponsor preference
- auto-balance
- outside leg preference

## Binary Volume

Adds commissionable volume from package purchases and renewals.

Volume can come from:

- package commissionable volume
- product volume
- service volume
- recurring renewal volume

## Binary Pairing Bonus

Pairs volume between left and right legs.

Configurable:

- pairing ratio
- commission percentage
- cycle period
- carryover enabled
- max payout cap
- rank requirement

## Binary Carryover

Stores unused volume per leg after a cycle closes.

## Ledger

Binary commissions are recorded in the reward ledger. Binary modules do not directly pay users.
